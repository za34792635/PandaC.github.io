Hibernate 是什么?

Hibernate 是 一个开源的 ORM 框架,对JDBC 进行轻量级的对象封装,实现 业务逻辑模型POJO 和 数据库表 之间的映射,提高读写数据库效率的”全自动”框架。

Hibernate 的优势和工作

Hibernate

  • 简化了数据库访问层繁琐的重复性代码，提供丰富的API 与数据库交换数据。
  • 使用了JAVA 反射机制，映射灵活性高。
  • 支持多种数据库拓展，提供多种表关联，事务管理，DQL,DML等操作。
  • 事务管理
  • 缓存配置

2 Hibernate 的 Configuration / SessionFactory/Session接口的工作

2.1 Configuration 接口 加载Xml配置文件

Configuation实例对象 : 主要用于启动,加载,管理hibernate 的配置文件信息。有以下常用方法:

public String getProperty(String propertyName):

首先获取并且加载 指定路径配置文件,等待Configuration对象 创建SessionFactory

2.2 SessionFactory 接口

SessionFactory接口负责 Hibernate 的初始化和 建立Session对象。起到了缓冲区的作用，有以下功能：

1）将自动生成的SQL语句，映射数据和可重复利用的数据放在 工厂。

2）保存了对数据库配置的所有 映射关系，

有以下特点：

1）它是线程安全的， 同一个实例能被多个线程和Session共享。

2）它是重量级的，一般一个项目只需要建立一个SessionFactory，因此我们可以使用工具类抽取出来。

public SessionFactory buildSessionFactory throws HibernateException

通过调用 Configuration 建造会话工厂,返回SessionFactory对象(当前会话工厂已知道 hibernate 核心配置的内容,”准备动手”)

示例：《建造一个工厂工具类直接生产Session》
