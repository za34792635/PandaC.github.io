ORM 思想与Java JPA

ORM 思想

ORM (Object relation Mapping ) 是实体类对象 与 数据库表 建立映射关系的思想,通过ORM 可以实现 操作对象即操作数据库, 无需过多地配置连接数据库,执行数据库指令和操作SQL的工作, 提高开发效率。

常用的ORM框架

市面是有常用的ORM 框架有Mybatis ,Myabtis Plus 和 Hibernate 等ORM 框架 。 ORM框架大部分都有以下功能：

  • 简化了数据库访问层繁琐的重复性代码，提供丰富的API 与数据库交换数据。
  • 使用了JAVA 反射机制，映射灵活性高。
  • 支持多种数据库拓展，提供多种表关联，DQL,DML等操作。
  • 事务管理
  • 缓存配置

Sun 公司提出的ORM思想规范 “JPA”

在基于ORM 框架中， Sun公司希望可以大一统地规范市面的 ORM框架，创建出了一套 JPA 规范接口。ORM框架都基于JPA规范接口和抽象类组件各自地实现自己的功能，减少开发者 各种ORM框架的学习成本。

JPA 规范

JPA （Java persistence API）是由sun公司提供一系列Java持久化接口，规范了ORM框架映射数据库结构的API

JPA 历史

作为EJB 3.0规范的一部分，Java持久API的第一个版本JPA 1.0于2006年发布。

以下是在JPA规范下发布的其他开发版本: -

  • JPA 2.0 - 此版本于2009年下半年发布。以下是此版本的重要功能: -

    • 它支持验证。
    • 它扩展了对象关系映射的功能。
    • 它共享缓存支持的对象。

  • JPA 2.1 - JPA 2.1于2013年发布，具有以下特性: -

    • 它允许提取对象。
    • 它为条件更新/删除提供支持。
    • 它生成模式。

  • JPA 2.2 - JPA 2.2在2017年作为维护开发而发布。它的一些重要特性是: -

    • 它支持Java 8的日期和时间。
    • 它提供了@Repeatable注释，当想要将相同的注释应用到声明或类型用法时可以使用它。
    • 它允许JPA注释在元注释中使用。
    • 它提供了流式查询结果的功能

JPA 规范了什么？

JPA 规范了 如果一个类 定义了JPA 规范， 则该对象称为JPA对象。

JPA规范了 JPA对象关系映射，JPA对象的实体类规范、实体管理、映射方式和类型、JPA 对象之间的关系（组合、继承、依赖等）

1.JPA实体规范

实体类:包含一个无参构造器的 POJO 类。

如何定义JPA实体类？

  • @Enttiy: 代表该类是一个JPA 实体类

  • @Tabel: 映射数据库表名。 默认与类名相同，如果不一致则通过@Table注解覆盖默认值

  • @Id : 代表该属性映射 是一个 表主键 属性 。默认与数据表的主键id一致。

    • @GeneratedValue:设置主键自增策略。

  • @Column : 代表该属性映射一个普通字段。默认与数据库表字段一致，如果不一致，使用@Column覆盖

    Tips:

    springboot 中JPA 默认自动将实体类中的驼峰命名转化为下划线隔开的方式进行数据库查询

  • @Embeddable :用于类中，代表该类是内嵌类,可嵌入到任何JPA类，通常用于定义数据表公共字段，如create_time , update_time , create_user ,update_user等等

    @Emdded:一般用于属性或方法, 代表是从外部嵌入的属性和方法。

到此 一个JPA 实体类定义完成。
